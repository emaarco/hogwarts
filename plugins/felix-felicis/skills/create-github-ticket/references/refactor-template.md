# Refactor Task Template

## Title format

refactor: <what is being restructured and why>

## Label

(none)

## Body sections

### Summary

Brief description of the refactoring task.

### Current State

Describe the current code structure or technical debt.

### Desired State

Describe the target state after refactoring.

### Added Value

Explain why this refactor is worth doing (maintainability, performance, clarity, etc.).

### Technical Notes (optional)

Affected files, dependencies, migration considerations, or implementation approach.
